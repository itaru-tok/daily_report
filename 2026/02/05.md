Q1. 以下のようなversions.tfの場合、1.12.0 を使うことは可能ですか？

```
terraform {
  required_version = "~> 1.11.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.93.0"
    }
  }
}
```

いいえ。1.11.0以上1.12.0未満


Q2. 以下のようなversions.tfの場合、1.12.0 を使うことは可能ですか？

```
terraform {
  required_version = "~> 1.11"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.93.0"
    }
  }
}
```
もちろん。1.11以上2未満

Q3. 以下のようなversions.tf、.terraform-versionの場合、バージョン1.12.0 へアップグレードするにはどんな修正が必要ですか？

```
versions.tf

terraform {
  required_version = "~> 1.11.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.93.0"
    }
  }
}
```
.terraform-version

`1.11.0`

required_version~> 1.12.0、.terraform-versionを1.12.0に変更し、terraform init -upgradeで実行

Q4. .terraform.lock.hclの役割はなんですか？
プロバイダーの依存関係やバージョンを固定し、チーム全体で同じプロバイダーを使用できるようにするため

Q5. .terraform.lock.hcl はどのタイミングで作成されますか？
terraform init

Q6. .terraform.lock.hcl はどんなコマンドを実行すると更新できますか？
terraform init -upgrate
